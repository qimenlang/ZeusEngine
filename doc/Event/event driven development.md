# 事件驱动开发

## 一、核心概念

1. ##### 概念：

   事件驱动开发（Event-Driven Development）是一种以事件为核心的编程范式，程序的**执行流程由外部事件**（如用户操作、系统通知、消息到达等）**触发**，而非传统的顺序控制流。

2. ##### 核心组件：

   ```graph
   graph LR
       A[事件源] -->|生成事件| B[事件队列]
       B -->|分发事件| C[事件处理器]
       C -->|执行操作| D[响应系统]
   ```

3. ##### 关键特征：

   - 异步处理：事件产生与处理解耦
   - *非阻塞I/O：高效利用系统资源*
   - 松耦合架构：组件间通过事件通信
   - 高并发支持：单线程处理大量事件
